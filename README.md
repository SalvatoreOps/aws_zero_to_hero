## рҹ“– What is AWS Zero to Hero?

A structured, self-paced learning journey designed to take you from **zero cloud experience** to confidently building on AWS. Each day focuses on a **single AWS service** вҖ” theory, hands-on practice, and a mini project.

No previous AWS experience needed. Just show up every day.

> **Challenge rule:** Spend at least 30 minutes per day exploring the service, build something small, and document what you learned. Consistency beats intensity.

### рҹ“… Week 0 вҖ” Before You Begin

| Day | Topic | Focus |

> рҹ’Ў **Day 0 goal:** Create your AWS account, explore the console, understand the big picture вҖ” then hit Day 1 with confidence.

<details>
<summary>рҹ“– <strong>Day 0 вҖ” What You'll Learn</strong></summary>

<br>

**What is Cloud Computing?**
- On-premise vs cloud vs hybrid models
- IaaS / PaaS / SaaS explained with AWS examples
- Why businesses move to the cloud (cost, scale, speed)

**AWS Global Infrastructure**
- **Regions** вҖ” independent geographic locations (e.g. `us-east-1`, `ap-south-1`)
- **Availability Zones (AZs)** вҖ” isolated data centres within a region
- **Edge Locations** вҖ” CDN endpoints used by CloudFront
- **Local Zones** вҖ” low-latency extension of a region

**AWS Core Service Categories**

| Category | Example Services |
|----------|-----------------|
| Compute | EC2, Lambda, ECS |
| Storage | S3, EBS, EFS |
| Database | RDS, DynamoDB, ElastiCache |
| Networking | VPC, Route 53, CloudFront |
| Security | IAM, WAF, Shield, KMS |
| DevOps | CodePipeline, CodeBuild, CodeDeploy |
| Monitoring | CloudWatch, X-Ray, CloudTrail |
| AI / ML | Bedrock, SageMaker, Rekognition |

**Account Setup Checklist**
- [ ] Create a new AWS account at [aws.amazon.com](https://aws.amazon.com)
- [ ] Enable MFA on the root account
- [ ] Set a billing alert at $5 in CloudWatch
- [ ] Explore the AWS Management Console
- [ ] Install and configure the AWS CLI v2
- [ ] Bookmark the [AWS Docs](https://docs.aws.amazon.com)

**Key Concepts to Know Before Day 1**
- **ARN** вҖ” Amazon Resource Name, a unique identifier for every AWS resource
- **Region vs AZ** вҖ” deploy across AZs for high availability
- **Shared Responsibility Model** вҖ” AWS secures the cloud; you secure what's *in* the cloud
- **Pay-as-you-go** вҖ” you only pay for what you use, when you use it

</details>
